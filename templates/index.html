What changed in the project

The system now tells one cleaner story from training to deployment.

01
Deep-learning-only deploymentThe handcrafted fallback classifier was removed and replaced with a TensorFlow Lite export from the real model.
02
Duplicate-aware evaluationValidation no longer benefits from exact-copy leakage across train and holdout.
03
Safer image handlingUnsupported uploads are rejected before diagnosis, which fixes the random-image problem.

Inference workflow

The user sees a simple surface while the backend does more careful work under the hood.

1
Decode and verifyUploads are decoded safely, validated as images, and size-checked before inference.
2
Filter unrelated inputsImage statistics are compared against the chest CT distribution to reject obvious non-CT content.
3
Run the deep modelTensorFlow is used for training, while deployment uses TensorFlow Lite from the same learned model.
4
Return structured outputThe response includes class, confidence, backend, and probability details in a clean payload.
